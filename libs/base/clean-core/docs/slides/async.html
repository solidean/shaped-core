How a cc::async node lives in memory
— and what runs when you sum two of them

The dataflow async is values, not futures. Each async<T, E> is a single cache-line node; a compute frame is polled to fill it in, and never blocks a thread. This deck traces one concrete graph byte by byte.

// the whole example this deck walks through
auto a = cc::make_async_lazy([]{ return 3; });                  // async<int>
auto b = cc::make_async_lazy([]{ return 4; });                  // async<int>
auto c = cc::make_async_lazy([](int x, int y){ return x + y; }, a, b); // c waits for a,b
int v = cc::async_blocking_get(c);                          // drives the graph -> 7

The shapeThree nodes, one dependency edge each

c is an async<int> whose value is the sum of two other async<int>s. Nothing computes until c is driven; then the graph pulls its dependencies in.

  • Lazy — a, b, c are born cold; requiring one schedules it.
  • Errors short-circuit — if a or b failed, c's function is skipped and the error propagates.
  • The handle owns the node — shared_async<int> is an 8-byte intrusive pointer; copying it never copies the node.
Requiring c schedules a and b; each resolves to a value, then c sums them.

The nodeOne async<int> = 64 bytes = one cache line

A 16-byte header followed by a payload slot. The node is cache-line aligned so unrelated nodes never share a line (they are polled and woken from different threads).

No separate control block: the strong/weak counts live inline at offset 0, so the handle is a single pointer. There is no C++ vtable — see the next slides for what replaces it.

The header, zoomed_state_and_ops: one word does four jobs

The 8-byte control word is a tagged pointer. Its high bits hold a 32-aligned async_type_ops*; the freed-up low 5 bits carry the lifecycle state, a wake flag, and the spinlock — so state, locking, and the type descriptor all read/write in one atomic.

The 3 state bits encode 7 states: cold → scheduled → running → blocked → ready_value / ready_error, plus external_pending for a manual/promise node. is_ready() is a lock-free acquire load of this word.

The payload slotOne 32-byte union, three lives

The offset-16 slot is a hand-managed union. Before the node is ready it holds the compute scratch; after, it holds the value or the error — never both, discriminated by the state bits. Value and error both start at offset 0 of the slot.

frame (unique_function, 1 ptr) pending deps continuations value (T) error (E)
The value/error grows the node for a large T/E (the slot is sized to the largest of scratch, T, E) — so async<int> and async<string> are one line, a big value spills onto more.

The hand-rolled vtableasync_type_ops — one shared static descriptor

The base node is type-erased (its poll loop names no T/E). To destroy the typed value or error and free the right size class, it follows the tagged pointer to a static-constexpr descriptor. It is keyed on (size class, value-teardown, error-teardown), not on (T, E) — so it collapses: a trivially-destructible type uses a null teardown, and async<int> and async<float> share the same descriptor pointer.

struct async_type_ops {
  void (*teardown_value)(node*);
  void (*teardown_error)(node*);
  node_class_index class_index;
};

No per-node vtable pointer, no virtual dtor — the descriptor lives in read-only static storage and the three nodes all point at the same one.

One descriptor, three referrers — and with the collapse, that same descriptor is reused by every async<trivially-destructible T, async_error> of this size class.

Where everything livesThe full allocation map for c = a + b

Driving the graph touches five arenas. Only the nodes and their frame closures are long-lived; the rest is transient scaffolding on the stack or in slabs.

The frame captures are what keep dependencies alive: c's closure holds strong handles to a and b. The pending-dep list owns nothing — it only tracks not-ready deps for scheduling.

The key moveHow a value overwrites the frame's own slot

When the poll loop runs a compute step it first moves the frame onto its own stack. That frees the slot, so finish_value can build the int straight into offset 16 under the node lock, then publish ready_value last.

  • Steal the continuation head (dependents to wake).
  • Tear down the unresolved scratch (frame shell + deps + conts).
  • Placement-new the value at offset 16.
  • Store ready_value (release) — a late reader never sees ready before the value.

If the frame instead parks (waiting/yield), it is moved back into the slot for the next poll — which is why the value may safely reuse its bytes.

Same 32 bytes, reinterpreted. The state bits are the discriminator.

The failure channelErrors are values in the same slot

A node completes with either a value or an E (default async_error), built into the very same offset-16 slot via finish_error — the mirror of finish_value. The ops descriptor's teardown_error destroys it.

  • High-level sugar auto-propagates: if a failed, c's (x, y) → x + y is skipped and the error flows to c.
  • A raw frame decides: it checks dep->try_error() and may transform, ignore, or propagate.

Read it zero-copy with c->try_error() → E const*, or move the whole outcome out with into_result(move(c)) → result<int, E>.

Same offset-16 start; the state (ready_value vs ready_error) says which arm is live.

Why it is shaped this wayThe load-bearing invariants

  • One cache line, no false sharing. The node is alignas(64); unrelated nodes on different threads never share a line.
  • One pointer per handle. Intrusive strong/weak counts at offset 0 mean no control block.
  • No vtable. A 32-aligned static async_type_ops tagged into the state word replaces it — state, lock, and type in one atomic.
  • Value grows the node, no inline cap. The slot is max(scratch, sizeof(T), sizeof(E)).
  • Polling never blocks. A frame that can't progress parks on not-ready deps and is woken; the eager depth-first drive completes most deps inline without ever subscribing.
  • Lost-wakeup-free. CAS state transitions: a dependency completing and scheduling a node can't be erased by that node parking itself.
  • Frame captures own deps. The pending list tracks; the closure retains.

The semantics and public API are the contract; this layout can change under the hood.

RecapThe 64 bytes, end to end

The one-liner: each async<T, E> is a cache-line node — a 16-byte intrusive header plus a union that is compute scratch before, value or error after. A type descriptor tagged into the state word stands in for a vtable. Driving the graph fills each node in place, depth-first, without ever blocking a thread.
